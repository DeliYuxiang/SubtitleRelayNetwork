# SRN 云端迁移与更新计划

本文件描述了将 SRN 中继从本地 Go/SQLite 架构迁移至 Cloudflare Serverless (Worker + D1 + R2) 架构的路线图。

## 第一阶段：基础设施搭设

1. **Worker 项目初始化**：
    - 使用 `wrangler` 创建 TypeScript 项目。
    - 使用 Web Crypto API 实现 Ed25519 协议签名验证。
2. **D1 元数据库初始化**：
    - 执行 `schema.sql`，创建包含 `content_md5`、`archive_md5` 等字段的新版 `events` 表。
3. **R2 存储桶配置**：
    - 创建用于存放字幕二进制文件的 R2 桶。
    - 配置相应的 CORS 或访问策略。

## 第二阶段：工具链升级 (Golang)

开发或升级 `cmd/migrate` 工具，使其支持 `cmd/cloud-sync` 功能：
1. **数据源扫描**：遍历本地 `srn.db` 的旧版事件。
2. **二进制提取**：将存放在 SQLite 里的字幕 blob 提取出来。
3. **云端同步逻辑**：
    - 计算文件的 `content_md5`。
    - 上传二进制内容到 **R2**。
    - 将包含 TMDB 索引和溯源链接的元数据推送到 **D1**。
    - 确保迁移后的 Event 签名依然符合去中心化协议标准。

## 第三阶段：客户端 (Hijarr) 适配

1. **解耦存储逻辑**：Hijarr 将不再假设中继节点有本地文件系统，而是完全依赖 RESTful API。
2. **聚合显示**：利用 `archive_md5` 在搜索结果中自动识别并合并同一字幕组的系列资源。
3. **快速查重**：在上传前，Hijarr 先通过文件 MD5 询问中继节点，若已存在则无需重复上传字节流。

## 第四阶段：安全与限速

1. **WAF 规则**：利用 Cloudflare 边缘防火墙过滤恶意爬虫和异常请求。
2. **多级限流**：
    - 基于 PubKey 的请求令牌桶算法（使用 KV 或 Durable Objects）。
    - 针对下载接口的边缘防盗链。
3. **动态令牌**：确保 R2 资源不公开，仅通过 Worker 生成的短效签名 URL 访问。

## 数据模型变更核心差异

| 字段 | 作用 | 来源 |
| :--- | :--- | :--- |
| `content_md5` | 核心去重指纹，防止存储膨胀。 | 扫描/上传时计算。 |
| `archive_md5` | 关联整季、整包资源，实现聚合预览。 | 从目录结构或压缩包名解析。 |
| `source_uri` | 记录原始出处（如磁力、RSS 链接等）。 | 采集器源数据。 |
| `r2_key` | 字幕在 R2 存储中的唯一定位键。 | 中继生成的路径。 |

**执行建议**：优先开发 `cmd/deploy-to-cloud` 逻辑，在本地保留 1.8GB 数据作为 Master，逐步推送到云端。
